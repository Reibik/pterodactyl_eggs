# 🦖 Pterodactyl Custom Eggs

Коллекция кастомных и глубоко доработанных шаблонов (Eggs) для панели управления **Pterodactyl**. 

Здесь собраны оптимизированные конфигурации серверов, в которые внедрены расширенные функции, QoL-улучшения и исправления логики запуска, недоступные в стандартных шаблонах. Репозиторий будет пополняться новыми играми.

---

## 📦 Доступные серверы

### 1. 🗿 SoulMask (Master Config)
Максимально прокачанный шаблон для хостинга SoulMask. Поддерживает работу как одиночных серверов, так и сложных кластерных связок из коробки.

**Ключевые особенности:**
* **🌐 Умный кластер (Cross-server):** Единый файл для `Main` и `Client` серверов. Достаточно выбрать роль сервера в настройках (0, 1 или 2), и панель сама сформирует нужные параметры (`-mainserverport`, `-clientserverconnect` и т.д.).
* **🔌 Встроенный RCON:** Полная поддержка RCON-команд для интеграции с автодонатом (например, WARGM) и внешними планировщиками. Порт и пароль задаются прямо в интерфейсе Pterodactyl.
* **🗺️ Выбор карты:** Удобное переключение между оригинальной картой (`Level01_Main`) и новой картой из дополнения (`DLC_Level01_Main`) без ручного редактирования строки запуска.
* **🔐 Динамический пароль:** Логический переключатель `Enable Password`. Если пароль выключен, флаг `-PSW` полностью вырезается из строки запуска, оставляя сервер открытым.
* **🛠️ PVE и лимиты:** Базовые параметры выживания (PVE-режим, лимиты гильдии) уже интегрированы в систему старта.

## ⚙️ Инструкция по установке (Как импортировать Egg)

1. Скачайте нужный файл `.json` из этого репозитория.
2. Авторизуйтесь в панели **Pterodactyl** с правами администратора (нажмите на иконку ⚙️ шестеренки в правом верхнем углу).
3. В левом навигационном меню перейдите в раздел **Nests** (Гнезда).
4. Выберите подходящее гнездо для игры (или создайте новое).
5. Нажмите кнопку **Import Egg** в правом верхнем углу.
6. Выберите скачанный `.json` файл и подтвердите загрузку.
7. Перейдите к вашему серверу, вкладка **Startup** (Запуск), и заполните появившиеся переменные под ваши нужды.

> **⚠️ Важное примечание по RCON и Кластерам:** Убедитесь, что дополнительные порты (для RCON или связи кластера) физически добавлены к вашему серверу в админ-панели на вкладке **Network** (Allocation Management), иначе Docker заблокирует к ним доступ.
